„On“, wenn die Datenbankverbindung beim Start einer Session geschlossen werden soll, andernfalls „Off“. Die Datenbankverbindung wird allerdings nur dann geschlossen, wenn die Option <tt>PAPAYA_DB_CONNECT_PERSISTENT</tt> in der conf.inc.php auf „FALSE“ gestellt ist. Diese Option soll die Performance von Webseiten erhöhen, die viele dynamische Requests enthalten. Das Öffnen einer Session kann sehr viel Zeit in Anspruch nehmen, da immer nur ein Request die Session öffen darf. Die Session bleibt somt für andere Requests gesperrt. Jeder folgende Request, der die Session-Daten anfordert, wird damit also blockiert. Damit die Datenbankverbindung für diesen Request aber nicht die ganze Zeit offen bleiben muss, können Sie sie mit dieser Option schließen. Damit kann die Datenbankverbindung von anderen Requests genutzt werden.
