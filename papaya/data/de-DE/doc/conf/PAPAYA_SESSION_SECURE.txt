"On", wenn Sessions für Frontend und Backend nur über HTTPS gestartet werden sollen, andernfalls "Off". Wenn Sie diese Option aktivieren, werden Session-Cookies für das Frontend nur über HTTPS-Verbindungen gesetzt. Session-Cookies werden dadurch auch nur für HTTPS gültig, sodass die Session-ID nicht über HTTP gesendet wird. Sie können die Einstellung „on“ zudem nur dann setzen, wenn Sie sich bereits über eine HTTPS-Verbindung im Backend angemeldet haben. Das Protokoll HTTPS kann auch exklusiv für das Backend erzwungen werden, siehe Parameter <tt>PAPAYA_UI_SECURE</tt>.
