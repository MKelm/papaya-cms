Domain for which the cookie is set. In the default settings for <tt>session.cookie_domain</tt>, no domain is given. Instead, the hostname of the server is used for the domain of the session cookie. But papaya CMS is capable do manage multiple subdomains. In order to prevent users from having to login again when changing the subdomain of the site, you can enter a wildcard, for example something like <code>domain.tld</code>.
